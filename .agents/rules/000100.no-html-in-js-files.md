**Keep HTML out of .js files**
* Elements that are created in js can be present on the page but hidden under a `display: none` or similar style directive. The goal is to make any code necessary for content available in the actual html/content pages, and leave the js file to scaffolding/manipulation/animation/AJAX only.
* Example:
```
    // DON'T DO THIS
    $actions.html(groups.length > jay.site.bfg.INITIAL ? `
        <button class="btn btn-outline js-toggle-bfg">
            <i class="fas fa-${jay.site.bfg.IS_EXPANDED ? 'minus' : 'plus'}"></i>
            ${jay.site.bfg.IS_EXPANDED ? 'Show Fewer Groups' : `Show All ${groups.length} Groups`}
        </button>
        <a href="https://forms.jaybaldwin.com/whatever" target="_blank" class="btn btn-primary">
            <i class="fas fa-users"></i> Find a Group Near You
        </a>
    ` : '');
```
* This code is hard to locate because it does not exist on the html page. this could be a div present on the page where the div is hidden/shown as needed, a preferred way to operate.
