## Change Coordination
When making changes:
1. **Identify scope**: List all files that might be affected
2. **Search references**: Use grep/search for resource names, variable names, output names
3. **Update systematically**: Make changes in dependency order
    * The code should always be formatted for human legibility
4. **Never delete whole blocks without permission**: rather, comment them out and note
`##### COMMENTED OUT BY __AGENT/PROGRAM NAME__ #####`
 at the top of the comment block
5. **Verify completeness**: Re-check all references after changes
