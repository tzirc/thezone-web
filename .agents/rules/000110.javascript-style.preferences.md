## JavaScript Style

All JavaScript files must follow the existing namespace/module pattern:

- **IIFE wrapper**: `(() => { ... })();`
- **Namespace registration**: `if (typeof(window.jay) === 'undefined') window.jay = {};` then `window.jay['moduleName'] = { ... };`
- **Sub-objects** group related state and methods (e.g. `nav`, `sidebar`, `gallery`)
- **State properties** use `ALL_CAPS` (e.g. `IS_EXPANDED`, `CURRENT_INDEX`)
- **Methods** are arrow functions assigned as object properties (e.g. `init: () => { ... }`)
- **Self-reference** uses the full namespace path — `jay.site.xxx` — never `this`
- **Each sub-object** exposes an `init()` method; the top-level `ready()` calls them all
- **jQuery** is preferred for all DOM operations
- **Top-level init**: `jay.moduleName.init();` calls a `.ready()` function: `init: () => { $(jay.moduleName.ready); }`
- **Entry point**: `jay.moduleName.init();` at the bottom of the IIFE

**Reference implementation**: `https://jaybaldwin.com/js/jay.gan.js` - although if I were writing this today, the entire body would be inside an IIFE.

**File naming convention**: `jay.<module-name>.js` (e.g. `jay.site.js`, `jay.analytics.js`)
